# VietActuaryKit

VietActuaryKit is a public Obsidian-compatible knowledge-base template for actuaries, data analysts, actuarial analysts, and AI coding/data agents working with Vietnam non-life insurance topics.

It includes reusable playbooks, data contracts, regulatory source references, privacy guardrails, app/SaaS engineering checklists, and agent operating rules. It is designed to be cloned as an Obsidian vault or copied into an existing vault.

## Who This Is For

- Actuarial analysts working on pricing, loss ratio, reserving, reporting, and data quality.
- Data analysts building reusable insurance analytics workflows.
- AI coding/data agents that need explicit guardrails before touching insurance or personal data.
- Builders creating internal tools, dashboards, or SaaS-style applications for insurance workflows.

## What Is Included

- Vietnam non-life insurance domain notes.
- Regulatory source register with official links.
- Personal data and privacy checklist.
- Pricing and loss ratio playbooks.
- Policy and claim data contract templates.
- App/SaaS product, architecture, API, database, auth/RBAC, security, and deployment playbooks.
- RTK token optimization guidance for coding agents.
- Project `AGENTS.md` template for repo-level agent instructions.

## What Is Not Included

This public template intentionally excludes private project knowledge, internal decision logs, company-specific pricing logic, internal workflows, secrets, API keys, credentials, customer data, claim data, policy data, and production configuration.

## How To Use

1. Clone this repository.
2. Open the folder as an Obsidian vault, or copy selected folders into your existing vault.
3. Start at `01_Dashboard/VietActuaryKit Home.md`.
4. For coding agents, copy or adapt `AGENTS.md` and `RTK.md` into your repo.
5. Replace placeholder paths such as `<YOUR_OBSIDIAN_VAULT_PATH>` and `<YOUR_CODEX_HOME>` with your own local paths only in your private copy.

## Important Disclaimer

This repository is a knowledge-base template for AI coding/data agents and actuarial/data workflows. It is not legal, actuarial, insurance, security, privacy, tax, financial, or compliance advice. Validate all regulatory content with qualified professionals, internal Legal/Compliance teams, and official sources.
